## Обзор системы

**VK Parser** — это комплексная система для анализа профилей ВКонтакте, написанная на Python. Система состоит из двух основных подсистем:

1. **Парсер профилей** — сбор, обработка и сохранение данных о пользователях VK
2. **Система идентификации** — сравнение профилей для определения того, могут ли два профиля принадлежать одному человеку

### Основные возможности

- Получение полной информации о пользователе (имя, дата рождения, город, страна, образование, работа)
- Загрузка списка друзей с детальной информацией
- Скачивание всех фотографий пользователя
- Сохранение данных в форматах JSON, CSV, TXT
- Генерация HTML-отчетов
- Сравнение профилей по 6 факторам с весовой системой
- Визуальное сравнение лиц (при наличии библиотек распознавания)

## Технологический стек

### Язык и платформа
- **Python 3.8+** — основной язык разработки
- **Flask** — веб-фреймворк для интерфейса (версия 2.3.0+)
- **Werkzeug** — WSGI библиотека (версия 2.3.0+)

### Работа с API и сетью
- **requests 2.28.0+** — HTTP-запросы к VK API
- **VK API v5.131** — официальный API ВКонтакте

### Обработка изображений и распознавание
- **Pillow 9.0.0+** — базовая обработка изображений
- **face_recognition 1.3.0+** — распознавание лиц на базе dlib (приоритетный метод)
- **OpenCV 4.5.0+** — каскады Хаара для детекции лиц (fallback)
- **MediaPipe** — решение от Google (fallback, отключено по умолчанию)
- **NumPy** — математические вычисления для face recognition

### Веб-интерфейс
- **HTML5/CSS3/JavaScript** — клиентская часть
- **Jinja2** — шаблонизатор Flask

## Поток данных

### Сценарий 1: Парсинг профиля через веб-интерфейс

```
1. Пользователь вводит URL профиля в форму
              │
              ▼
2. POST /parse → web/app.py:parse()
              │
              ▼
3. Создание session_id, запуск асинхронного потока
              │
              ▼
4. parse_profile_async() выполняет:
   ├── VKApiClient.get_user_info() → получение данных пользователя
   ├── FriendsHandler.get_all_friends() → загрузка списка друзей
   ├── PhotoHandler.get_all_photos() → получение списка фото
   ├── PhotoHandler.download_photos() → скачивание фото на диск
   └── FileExporter.save_json() → сохранение в results/
              │
              ▼
5. Сохранение в profiles_cache[session_id]
              │
              ▼
6. GET /parse/result/<session_id> → отображение результатов
```

### Сценарий 2: Сравнение профилей

```
1. Пользователь выбирает два профиля (URL или сохраненные)
              │
              ▼
2. POST /compare → web/app.py:compare()
              │
              ▼
3. Загрузка данных профилей (из файлов или через API)
              │
              ▼
4. ProfileComparer.compare_profiles() выполняет:
   ├── NameMatcher.compare_full_names() → сравнение имен (15%)
   ├── GeoMatcher.compare_locations() → геолокация (10%)
   ├── FriendsMatcher.compare_friends() → друзья (25%)
   ├── ContentMatcher.compare_interests() → контент (15%)
   ├── DemographicsMatcher.compare_all_demographics() → демография (10%)
   └── VisualMatcher.compare_avatars() → лица (25%)
              │
              ▼
5. Расчет взвешенной оценки + бонусы
              │
              ▼
6. GET /comparison/<id> → отображение результатов сравнения
```

## Модули и их назначение

### 1. VK API Client (`src/vk_api/vk_api_client.py`)

**Класс:** `VKApiClient`

**Назначение:** Централизованная работа с VK API v5.131

**Основные методы:**
- `get_user_info(user_input)` — получение данных пользователя по URL, ID или screen_name
- `resolve_screen_name(screen_name)` — преобразование короткого имени в ID

**Особенности:**
- Автоматические повторные попытки при ошибках сети (до 3 раз)
- Обработка ошибок API:
  - Код 5: Недействительный токен
  - Код 6: Превышен лимит запросов (rate limit)
  - Код 18: Пользователь удален
  - Код 30: Профиль приватный
- Логирование всех запросов через модуль `logger`

**Запрашиваемые поля пользователя:**
```python
fields = [
    'id', 'first_name', 'last_name', 'sex', 'bdate', 'city', 'country',
    'photo_max', 'photo_max_orig', 'has_photo', 'online', 'domain',
    'about', 'activities', 'books', 'games', 'interests', 'movies', 'music',
    'career', 'military', 'education', 'universities', 'schools',
    'occupation', 'personal', 'relatives', 'home_town', 'status',
    'followers_count', 'counters', 'instagram', 'facebook'
]
```

### 2. Photo Handler (`src/handlers/photo_handler.py`)

**Класс:** `PhotoHandler`

**Назначение:** Работа с фотографиями пользователя

**Основные методы:**
- `get_all_photos(user_id)` — получение всех фотографий (постранично, до 200 за запрос)
- `download_photos(photos_list, save_dir, user_name)` — скачивание фото на диск
- `_get_best_photo_url(photo)` — поиск URL максимального размера

**Лимиты:**
- Максимум 200 фото за запрос (`MAX_PHOTOS_PER_REQUEST`)
- Задержка 0.3 сек между запросами страниц
- Задержка 0.5 сек после каждых 20 скачанных фото

### 3. Friends Handler (`src/handlers/friends_handler.py`)

**Класс:** `FriendsHandler`

**Назначение:** Загрузка и анализ списка друзей

**Основные методы:**
- `get_all_friends(user_id, max_count=5000)` — получение всех друзей (до 5000 за запрос)
- `analyze_friends_stats(friends_list)` — статистика по друзьям (пол, возраст, города)

**Лимиты:**
- Максимум 5000 друзей за запрос
- Максимум 5000 друзей всего (`max_count`)
- Задержка 0.3 сек между запросами

### 4. File Exporters (`src/handlers/file_exporters.py`)

**Класс:** `FileExporter`

**Назначение:** Экспорт данных в различные форматы

**Методы:**
- `save_json(data, filepath)` — сохранение в JSON
- `save_csv(data, filepath)` — сохранение в CSV
- `save_txt(data, filepath)` — сохранение в текстовый файл

### 5. Data Preparer (`src/utils/data_preparer.py`)

**Класс:** `DataPreparer`

**Назначение:** Подготовка и нормализация данных для экспорта и сравнения

**Методы:**
- `prepare_user_data(user)` — подготовка данных пользователя
- `prepare_friends_data(friends)` — подготовка данных о друзьях

## Система идентификации профилей

### Главный оркестратор (`src/matchers/profile_comparer.py`)

**Класс:** `ProfileComparer`

**Назначение:** Координация всех модулей сравнения и вычисление итоговой вероятности

### Весовая система факторов

| Фактор | Вес | Модуль | Описание |
|--------|------|--------|----------|
| **Имя** | 15% | `name_matcher.py` | Точное, нечеткое, фонетическое сравнение |
| **Фото (визуальный)** | 25% | `visual_matcher.py` | Сравнение лиц, коллекции фото |
| **Друзья** | 25% | `friends_matcher.py` | Общие друзья, коэффициент Жаккара |
| **Контент** | 15% | `content_matcher.py` | Интересы, стиль, тексты |
| **Геолокация** | 10% | `geo_matcher.py` | Город, страна, родной город |
| **Демография** | 10% | `demographics_matcher.py` | Возраст, пол, образование |

### Формула расчета

```
final_score = Σ(weight_i × score_i) + bonuses
final_percentage = min(100, max(0, final_score × 100))
```

Где `bonuses` начисляются за:
- Точное совпадение имени (≥90%) — +15%
- Общие друзья ≥20 — +15%, ≥10 — +10%, ≥5 — +5%
- Точная геолокация (≥95%) — +10%
- Идентичные фото ≥3 — +20%, ≥1 — +10%

### Интерпретация результатов

| Оценка | Интерпретация |
|--------|----------------|
| ≥90% | 🟢 ПРАКТИЧЕСКИ ТОЧНО — один человек |
| ≥75% | 🟢 ВЫСОКАЯ ВЕРОЯТНОСТЬ — очень вероятно один человек |
| ≥60% | 🟡 СРЕДНЯЯ ВЕРОЯТНОСТЬ — вероятно один человек |
| ≥40% | 🟠 НИЗКАЯ ВЕРОЯТНОСТЬ — маловероятно |
| ≥20% | 🔴 ОЧЕНЬ НИЗКАЯ ВЕРОЯТНОСТЬ — скорее всего разные |
| <20% | 🔴 ПРАКТИЧЕСКИ НЕВОЗМОЖНО — разные люди |

## Алгоритмы сравнения

### 1. Name Matcher (`src/matchers/name_matcher.py`)

**Методы сравнения имен:**
- **Точное совпадение** — после нормализации (удаление пробелов, нижний регистр)
- **Нечеткое сопоставление** — расстояние Левенштейна, коэффициент SequenceMatcher
- **Фонетический анализ** — алгоритмы Soundex и Metaphone для русского языка
- **Вариации имен** — словарь сокращений и уменьшительных форм (Даниил → Даня, Данил, Данила)
- **Транслитерация** — преобразование между кириллицей и латиницей (Данил ↔ Daniil)

**Пример словаря вариаций:**
```python
NAME_VARIATIONS = {
    'даниил': ['даниил', 'данил', 'данила', 'данилка', 'даня', 'данек'],
    'александр': ['александр', 'алексей', 'алекс', 'саша', 'саня'],
    'мария': ['мария', 'маша', 'машенька', 'мари', 'мотя'],
    # ... еще 15+ имен
}
```

### 2. Visual Matcher (`src/matchers/visual_matcher.py`)

**Используемые инструменты (в порядке приоритета):**

#### FaceRecognitionEngine (приоритет №1)
- Библиотека: `face_recognition` (на базе dlib)
- Алгоритм: нейросеть ResNet-34 для создания 128-мерных векторов лиц
- Сравнение: евклидово расстояние между векторами
- Порог совпадения: `FACE_MATCH_THRESHOLD = 0.6`

```python
# Пример работы
encoding1 = face_recognition.face_encodings(image1)[0]
encoding2 = face_recognition.face_encodings(image2)[0]
distance = np.linalg.norm(encoding1 - encoding2)
is_match = distance < 0.6
similarity_percentage = max(0, (1 - distance / 1.5)) * 100
```

#### OpenCVFaceRecognizer (fallback)
- Библиотека: `cv2` (OpenCV)
- Алгоритм: Haar Cascade Classifier (`haarcascade_frontalface_default.xml`)
- **Ограничение:** только обнаружение лиц, НЕ сравнение (возвращает similarity=50%)

#### MediaPipeFaceRecognizer (fallback, отключен)
- Библиотека: `mediapipe` от Google
- Статус: требует ручной загрузки моделей (`.blob` файлы)
- В коде отключен: `self.is_available = False` (строка 802)

**Процесс сравнения аватарок:**
1. Извлечение URL аватарок из данных пользователя (`photo_max`, `photo_max_orig`)
2. Скачивание аватарок во временную папку `temp_avatars/`
3. Попытка сравнения через face_recognition
4. Fallback на OpenCV при недоступности face_recognition
5. Удаление временных файлов

### 3. Friends Matcher (`src/matchers/friends_matcher.py`)

**Методы анализа друзей:**
- **Коэффициент Жаккара:** `J(A,B) = |A∩B| / |A∪B|`
- **Процент от каждого списка:** `common/total_1` и `common/total_2`
- **Анализ демографии друзей:** пол, возраст, города

**Пример расчета:**
```python
ids1 = {1, 2, 3, 4, 5}
ids2 = {3, 4, 5, 6, 7}
common = ids1 & ids2 = {3, 4, 5}  # 3 общих
jaccard = 3 / 7 ≈ 0.43  # Коэффициент Жаккара
```

### 4. Geo Matcher (`src/matchers/geo_matcher.py`)

**Методы сравнения:**
- Нормализация названий городов (Москва = Мск = Moscow)
- Сравнение стран
- Учет родного города (`home_town`)
- Часовые пояса

### 5. Content Matcher (`src/matchers/content_matcher.py`)

**Анализируемые данные:**
- Интересы (`interests`)
- Статус (`status`)
- О себе (`about`)
- Деятельность (`activities`)
- Музыка, фильмы, книги, игры, цитаты
- Посты пользователя

**Методы:**
- Пересечение множеств интересов
- Анализ стиля написания (формальность, экспрессивность)

### 6. Demographics Matcher (`src/matchers/demographics_matcher.py`)

**Сравниваемые факторы:**
- Дата рождения (возраст)
- Пол (`sex`: 1 — женский, 2 — мужской)
- Образование (университеты, школы)
- Карьера (карьерные данные)
- Семейное положение (`relation`)

```python
REQUEST_TIMEOUT = 10           # Таймаут запроса (сек)
MAX_RETRIES = 3                # Максимум попыток
RETRY_DELAY = 1                # Задержка между попытками (сек)

# Логирование
LOG_LEVEL = os.environ.get('LOG_LEVEL', 'INFO')
LOG_FILE = os.environ.get('LOG_FILE', 'vk_parser.log')

# Временные файлы
TEMP_FOLDER = "temp_avatars"
```

### Получение токена VK API

2. Создать новое приложение (тип: Standalone)
3. Раздел "Настройки" → "Ключи доступа"
4. Скопировать `access_token`

**Важно:** Токен должен иметь права доступа: `friends`, `photos`, `status`, `wall`, `groups`.

## Обработка ошибок

### Типы ошибок

#### 1. Ошибки VK API

| Код | Описание | Действие |
|-----|----------|----------|
| 5 | Недействительный токен | Проверка токена, запрос нового |
| 6 | Превышен лимит запросов (3/сек) | Повторная попытка с задержкой |
| 18 | Пользователь удален/заблокирован | Прекращение обработки |
| 30 | Профиль приватный | Частичная обработка доступных данных |

#### 2. Ошибки сети

- **Timeout** — превышение `REQUEST_TIMEOUT` (10 сек)
- **ConnectionError** — потеря соединения
- **RequestException** — прочие ошибки requests

**Стратегия:** до `MAX_RETRIES` (3) попыток с увеличивающейся задержкой (`RETRY_DELAY * (retry_count + 1)`)

#### 3. Ошибки данных

- Отсутствующие поля в ответе API
- Неверный формат данных
- Несуществующие файлы

**Обработка:** использование `.get()` с значениями по умолчанию, try-except блоки

### Логирование

**Модуль:** `src/utils/logger.py`

**Уровни:**
- `DEBUG` — детальная информация для отладки
- `INFO` — подтверждение успешных операций
- `WARNING` — предупреждения (например, rate limit)
- `ERROR` — ошибки (недействительный токен, сетевые сбои)

**Пример использования:**
```python
logger = get_logger(__name__)
logger.debug("API request: users.get")
logger.info("Пользователь найден: Иван Иванов")
logger.warning("API error 6: слишком много запросов")
logger.error("Токен недействителен")
```

## Безопасность

### Рекомендации

#### 1. Хранение токена
- ✅ Использовать переменные окружения
- ✅ Использовать файл `token.txt` (добавить в `.gitignore`)
- ❌ НЕ коммитить токен в репозиторий
- ❌ НЕ вписывать токен прямо в код

#### 2. Ограничение доступа
- Не хранить персональные данные дольше необходимого
- Использовать шифрование при передаче данных (HTTPS в production)
- Ограничить доступ к папке `results/`

#### 3. Правовые аспекты
- Соблюдать [Terms of Service VK](https://vk.com/terms)
- Использовать только для легальных целей
- Не распространять полученные персональные данные без согласия

#### 4. Веб-безопасность (Flask)
- В production отключить `debug=True`
- Использовать `secret_key` из переменных окружения
- Валидировать входные данные (URL профилей)
- Ограничить размер загружаемых файлов

## Глоссарий

| Термин | Описание |
|--------|----------|
| **VK API** | Программный интерфейс ВКонтакте для получения данных |

| **Levenshtein distance** | Расстояние редактирования между строками |
| **Soundex** | Фонетический алгоритм для индексации звуков речи |
| **Metaphone** | Улучшенный фонетический алгоритм |
